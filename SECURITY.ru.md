# Политика безопасности

[English](./SECURITY.md) | [Русский](./SECURITY.ru.md)

## Автоматическое сканирование безопасности

HvostID использует три автоматических уровня для обнаружения уязвимостей
и управления ими:

| Инструмент | Область | Триггер | Порог срабатывания |
|------------|---------|---------|--------------------|
| **Dependabot** | Зависимости Gradle, npm, Docker, GitHub Actions | Еженедельно | Открывает PR |
| **OWASP Dependency Check** | CVE в транзитивных зависимостях Java и npm | Каждый PR | CVSS ≥ 7.0 (high/critical) |
| **Trivy** | Слои Docker-образов и пакеты ОС | Каждый мерж в `main` | Severity CRITICAL |

Результаты сканирования Trivy загружаются в формате SARIF и видны во
вкладке **Security → Code scanning** репозитория.

## Подавление ложных срабатываний

Известные ложные срабатывания для OWASP Dependency Check задокументированы
в [`dependency-check-suppressions.xml`](./dependency-check-suppressions.xml).
Каждая запись подавления должна содержать элемент `<notes>` с объяснением,
почему находка неприменима.

Чтобы добавить подавление:

1. Идентифицируйте CVE и убедитесь, что это ложное срабатывание
   (неверный CPE, недостижимый код и т.д.).
2. Добавьте запись в `dependency-check-suppressions.xml`:
   ```xml
   <suppress>
       <notes>CVE-XXXX-XXXXX неприменима, потому что ...</notes>
       <cve>CVE-XXXX-XXXXX</cve>
   </suppress>
   ```
3. Откройте PR с подавлением и обоснованием в описании PR.
   Техлид должен его одобрить.

## Сообщение об уязвимости

Если вы обнаружили уязвимость в HvostID, пожалуйста, **не открывайте
публичный GitHub issue**.

**Шаги:**

1. Откройте [GitHub Security Advisory](https://github.com/hvostid/hvostid/security/advisories/new)
   в этом репозитории (по умолчанию приватный).
2. Опишите уязвимость: затронутый компонент, шаги воспроизведения,
   потенциальное воздействие.
3. Техлид подтвердит получение отчёта в течение **2 рабочих дней**
   и предоставит первоначальную оценку в течение **5 рабочих дней**.

## Процесс реагирования

| Шаг | Ответственный | SLA |
|-----|---------------|-----|
| Подтверждение отчёта | Техлид | 2 рабочих дня |
| Оценка severity (CVSS) | Техлид | 5 рабочих дней |
| Разработка и тестирование исправления | Ответственный разработчик | Зависит от severity: Critical — 3 дня, High — 7 дней, Medium/Low — следующий спринт |
| Деплой исправления в `main` | Техлид | После ревью и мержа |
| Закрытие advisory | Техлид | После деплоя |

## Политика обновления зависимостей

- **Dependabot PR** для patch и minor обновлений группируются и должны
  быть проверены и замержены в течение одной недели.
- **Обновления major версий** обрабатываются вручную — создаётся
  отдельная задача в бэклоге.
- Если Dependabot PR исправляет известную CVE, он обрабатывается как
  исправление безопасности и мержится приоритетно.
